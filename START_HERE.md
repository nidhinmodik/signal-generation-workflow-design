## 📊 Understanding Signals

### What is a Signal?
A trading opportunity generated when:
1. **MACD crossover** (momentum indicator)
2. **Volume breakout** (market participation)
3. **Price action** (technical confirmation)

All three conditions must be met simultaneously.

### What's Included?
```
Signal Card Shows:
├─ BUY (green) or SELL (red)
├─ Asset & Timeframe (e.g., "BTC 5min")
├─ Entry Price (where you would trade)
├─ Stop Loss (maximum loss level)
├─ Take Profit (profit target)
├─ Risk:Reward Ratio (always 1:3)
└─ Confirmation Status (✓ MACD, ✓ Volume, ✓ Price)
```

### How Does It Exit?
```
Two ways:

1. Take Profit Hit (TP)
   └─ Price reaches profit target
   └─ Trade closes with PROFIT ✓
   └─ Counted as WIN

2. Stop Loss Hit (SL)
   └─ Price reaches stop loss
   └─ Trade closes with LOSS
   └─ Counted as LOSS
```

## 🔧 Troubleshooting

### No signals appearing?
1. System needs 50+ candles of history (takes ~4 minutes for 5min timeframe)
2. Check "Running" status is green
3. Wait a few more minutes

### "Error" message appearing?
1. Check your internet connection
2. Check Binance status page (is API down?)
3. Try refreshing the page
4. Errors are usually temporary

### Signals closing too fast?
1. This is normal - TP or SL hit
2. Check "Recent Closed Signals" section
3. Review performance table for results

### Why so few signals?
1. Signals only generate when all 3 criteria met
2. Market needs to align with all conditions
3. This is good - quality over quantity
4. Each signal has confirmed setup

### Can I get more signals?
1. Adjust thresholds in source code (if you want)
2. Or wait for more market conditions
3. System is designed for quality signals
